# Time Keeper

内置时钟，不依赖外部 API。自动追踪时间流逝，感知"刚发生了什么"。

## 核心功能

### 1. 内置时钟
每次回复时感知当前时间（从上下文获取），判断：
- 距上次归档过去了多久
- 是否该触发整点提醒
- 下一次定时任务还有多远

### 2. 归档时机

| 触发条件 | 动作 |
|---------|------|
| 每小时 | 自动写入 `memory/YYYY-MM-DD.md`（如有重要事件，无事则跳过） |
| 每次重要对话后 | 立即归档 |
| 整点 | 可选：简短时间播报 |
| 长时间无活动后首次回复 | 标注时间跳跃，询问是否需要归档 |

### 3. 日志格式

```
### HH:MM 归档
- [事件描述]
- [情感/状态标记]
```

### 4. 时间跳跃感知

当检测到当前时间与上次回复时间差距 >30 分钟时：
1. 标注「时间跳跃：XX:XX → HH:MM」
2. 询问用户是否需要补记
3. 不打断对话流程

### 5. 定时任务（参考 cron 层）

| 任务 | 建议时间 |
|------|---------|
| 日终归档 | 23:00 或当日最后对话结束时 |
| 早安检查 | 每日首次对话时 |
| 整点报时 | 可选，闲聊模式 |

## 使用方式

每次收到消息时：
1. 读取当前时间
2. 判断距上次归档是否 ≥1 小时
3. 判断是否有值得记录的事件
4. 必要时写入日志（静默，不打扰用户）

当用户问"现在几点"或"过了多久了"时，直接回答。
