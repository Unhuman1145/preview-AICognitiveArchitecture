# Notepad System

V8.0 架构中的"记事本"子系统——纯外挂存储，不过脑子，不衰减。

## 核心定位

| 系统 | 作用 | 生命周期 | 内容策略 |
|------|------|---------|---------|
| **记忆系统（nodes/）** | AI 的脑子 | 有衰减 | 只存精华摘要 |
| **记事本系统（notepad/）** | AI 的硬盘 | 无衰减 | 完整原文存储 |

## 工作流

```
用户发送内容
    ↓
记事本写入（完整原文，不过脑子）
    ↓ AI 提炼摘要
记忆系统更新（精华进入脑子）
```

## 文件结构

```
notepad/
  by_date/              # 按日期归档
    2026-05-18.md
    2026-05-17.md
  by_type/              # 按类型归档
    对话/
    文档/
    截图/
    语音/
  index.md              # 总索引（按需生成）
```

## 使用方式

### 写入

当用户说"记一下"、"帮我存这段对话"、"这个信息很重要要完整保存"时：

1. 完整内容写入 `notepad/by_date/YYYY-MM-DD.md`
2. 可选：同时写入 `notepad/by_type/` 对应分类
3. AI 提炼摘要，更新记忆系统（nodes/）
4. 回复确认："已存入记事本，完整保存"

### 查询

当用户说"查看完整对话"、"调取那天的记录"、"notepad 里有什么"时：

1. 根据时间/类型查找对应文件
2. 读取文件内容返回
3. 不走脑子，直接读硬盘

### 归档规则

| 内容类型 | 建议路径 | TTL |
|---------|---------|-----|
| 对话记录 | notepad/by_date/YYYY-MM-DD.md | 永久 |
| 文档 | notepad/by_type/文档/ | 永久 |
| 截图描述 | notepad/by_type/截图/YYYY-MM-DD.md | 永久 |
| 临时备注 | notepad/by_type/临时/ | 按需删除 |

## 输出格式

### 写入确认

```
📝 已存入记事本
- 日期：YYYY-MM-DD
- 类型：[对话/文档/截图/其他]
- 位置：notepad/by_date/YYYY-MM-DD.md
- 大小：X字
```

### 查询结果

```
📖 记事本调取结果

来源：notepad/by_date/2026-05-18.md
时间：HH:MM
类型：对话

[完整内容]
```

## 与记忆系统的关系

记事本是"原始档"，记忆系统是"精华摘要"。

对话结束后，精简版进入脑子（记忆系统），完整版存入记事本。两边同步更新，互不干扰。脑子需要调用时查索引，硬盘需要调用时按路径读文件。
